Overview

This package provides a complete set of files for a Progressive Web App that gets a perfect score from Google Lighthouse for a PWA. Use these files as a basis for building your own PWA.

For the package to work perfectly for you right out of the gate it must be run:

  1. Under its own domain or sub-domain
  2. Using HTTPS (SSL/TLS)
  3. On any modern web server (Apache, Nginx, Node.js, Python, etc.)

The app.js file loaded by index.html registers a service worker and performs advanced connectivity detection using modern web APIs and multiple test endpoints.

Connectivity Detection
The app uses a robust connectivity detection system that tests multiple endpoints including Google's favicon, HTTPBin, GitHub API, and local assets. It also leverages the Network Information API when available to provide detailed connection quality information including network type, speed, and data saver status.

Service Worker
The sw.js service worker file implements advanced caching strategies to ensure the PWA can run offline. It uses cache-first, network-first, and stale-while-revalidate strategies depending on the resource type. When connectivity cannot be established, users are redirected to a custom offline page with helpful information.

Caching Configuration
To modify which files are cached, update the staticAssets array in the sw.js file. To force cache updates and deploy new versions, increment the cacheName constant in sw.js.

Web Manifest
The manifest.webmanifest file is a JSON file containing PWA configuration settings:

  • short_name - Short app name for home screen
  • name - Full application name
  • scope - Navigation scope for the PWA
  • start_url - Initial URL when launched
  • background_color - Background color during launch
  • display - Display mode (standalone, fullscreen, etc.)
  • theme_color - Theme color for UI chrome
  • icons - App icons for different sizes and contexts
